## 1. 一句话讲清产品

**ZeroCode 是一个统一的 AI API 中转站。** 一把 Key，直连 Claude / GPT / Gemini 三大家族的主流模型；人民币充值、按量扣费；Cursor、Claude Code、Codex、Cline、Continue 等主流 AI 编程工具全部兼容，**改一行环境变量**就能用。

目标用户：**不想折腾国外信用卡、不想自己搞代理、希望一个地方解决所有模型调用**的独立开发者和 AI 重度用户。

## 3. 产品能力清单

### 3.1 兼容的 API 协议

- **Anthropic 原生**：`/v1/messages`、`/v1/messages/count_tokens`
- **OpenAI 原生**：`/v1/chat/completions`、`/v1/responses`（Responses API 支持 WebSocket）
- **Google Gemini 原生**：`/v1beta/models/{model}:generateContent` 等全套
- **流式输出（SSE）**：全部协议原生支持，对话应用无感知

### 3.2 模型覆盖

| 家族 | 代表模型 |
|------|---------|
| Claude | Opus 4.6 / Opus 4.6 Thinking / Sonnet 4.6 / Sonnet 4.5（含 thinking 变体）/ Haiku 4.5 |
| GPT | GPT-4 Turbo / GPT-4o / o1 全系（含 Priority 与 Flex 两档 SLA） |
| Gemini | 2.5 Flash / 2.5 Pro / 3 Flash / 3 Pro / 3.1 Pro（含 image 变体） |
| AWS Bedrock | 跨区 Claude Opus/Sonnet（US / EU / APAC 区域前缀自动适配） |
| 其他 | gpt-oss-120b 等开源模型 |

> 具体支持清单会随上游变动，现场报价前请以管理后台「模型定价」页为准。

### 3.3 支持的客户端（常见组合速查表）

| 客户端 | 协议 | 接入方式 |
|--------|------|----------|
| Claude Code（官方 CLI） | Anthropic | `export ANTHROPIC_BASE_URL=https://zerocode.kaynlab.com` + `export ANTHROPIC_AUTH_TOKEN=<你的 Key>` |
| Cursor | OpenAI | 设置 → Models → 填 Base URL + API Key |
| Codex CLI | OpenAI | `openai.base_url` 替换 |
| Cline / Roo / Continue | OpenAI / Anthropic 任选 | 填 Base URL + Key |
| Gemini CLI / Google AI SDK | Gemini | `GOOGLE_API_ENDPOINT` 替换 |

## 4. 用户使用流程（从零到跑通五步）

1. **注册**：邮箱注册 / LinuxDO 一键登录 / OIDC 企业登录。可选开启 TOTP 二次验证。
2. **充值**：后台「余额」→ 支付宝 / 微信扫码 / Stripe 刷卡。余额按美元记账，界面同时显示人民币折算。
3. **建 Key**：后台「API Keys」→ 新建，可给每把 Key 单独设美元额度上限（防一把 Key 被滥用跑爆总余额）。
4. **接入客户端**：把 `BASE_URL` 和 `API Key` 填进 Claude Code / Cursor 等工具对应字段（上表 §3.3）。
5. **看用量**：Dashboard 分模型、分天看请求数、token、费用、平均延迟；7 天趋势图；每条请求明细可查。

## 6. 常见问题 FAQ

### 售前

**Q：和直接买官方 API 有什么不一样？**
A：三点——支持人民币直付、一把 Key 同时支持三家模型、我们帮你扛上游抽风。

**Q：服务稳不稳定？**
A：每个模型组后面有多个上游账号池，任何一个被限流 / 短时故障，系统秒级切换到其他账号，客户端感知不到。

**Q：会不会跑路？**
A：支付全程走微信支付 / 支付宝 / Stripe 官方通道，不走任何二级账户；线上运营地址公开，可查 ICP / 注册主体。余额随时可查。

**Q：可以开发票 / 退款吗？**
A：▢ 【销售按本月政策填写】

### 接入

**Q：Claude Code 能用吗？怎么配？**
A：原生支持，两条命令：
```bash
export ANTHROPIC_BASE_URL=https://zerocode.kaynlab.com
export ANTHROPIC_AUTH_TOKEN=sk-<你的 Key>
```
然后正常 `claude` 就行。

**Q：Cursor 呢？**
A：设置 → Models → OpenAI API Key 填我们的 Key，Base URL 填 `https://zerocode.kaynlab.com/v1`，模型名填你要用的（如 `claude-sonnet-4-6`）。

**Q：一把 Key 能不能同时给多个工具用？**
A：可以。但建议给每个工具单独开一把 Key，设不同额度，方便管理和出了问题单独禁用。

### 计费

**Q：余额怎么计算？**
A：充值后以美元余额记账，每次请求按模型单价实扣。Dashboard 可以看到每条请求的原价（标准价）和实付价（叠加缓存折扣后的价）。

**Q：缓存命中怎么算便宜？**
A：Claude 支持 Prompt Caching——系统提示词、长文档这种重复传的内容，第二次起走"缓存命中价"，便宜很多。多轮对话越长越划算。

**Q：怎么防止一把 Key 被人盗刷？**
A：每把 Key 可以单独设美元额度上限，到顶自动停，不影响其他 Key 和主余额。

### 稳定性

**Q：突然连不上怎么办？**
A：先看后台「公告」和「用量」里的错误记录。一般自动切换已经处理掉大部分问题；若持续异常，联系客服。

**Q：限速吗？**
A：按账号等级和 Key 级别有并发数和 QPS 软限制，正常个人使用远低于这个阈值。重度用户可申请提升。

### 安全

**Q：请求内容会被保存吗？**
A：**不保存请求和响应正文**。系统只记录用量元数据（模型名、token 数、耗时、费用），用于计费和用量统计。

**Q：支持二次验证吗？**
A：支持 TOTP（Google Authenticator / 1Password 等），账户设置里可开启。
