# DEBUG REPORT: Claude-GPT long-context empty replies

- **Symptom:** Production Claude-GPT bridge conversations start normally, then after the history grows they repeatedly show thinking/reasoning followed by no assistant text or an empty response. The reported production user was `gybilly2023@gmail.com`.
- **Root cause:** Claude Code's hidden compact request is a high-confidence trigger, but compression alone is not the direct outage. When compact exceeded the upstream context window or ended as `response.failed`, incomplete/missing-terminal, or reasoning-only output, the bridge had already emitted Anthropic preamble/thinking and then normalized the terminal state to `message_stop/end_turn`. That converted a failed compact into a successful empty reply, prevented account failover, and left the client carrying the same uncompressed history into the next turn.
- **Contributing causes:** The API-key replay guard could discard older history before recovery; some visible text/tool arguments existed only in terminal `response.output`; recovery requests could inherit session/turn-state headers; and a `store=false` recovery response could be incorrectly reused as `previous_response_id`.
- **Fix:** Buffer non-visible SSE events until visible text/tool output exists; classify failed/incomplete/missing-terminal outcomes before writing downstream; replay terminal output with a complete Anthropic SSE lifecycle; keep a full transcript snapshot; and recover compact requests through bounded chunk summaries, recursive overflow splitting, hierarchical merge, model fallback, and an emergency local capsule. Recovery clears stateful headers, accumulates usage across all attempts, skips continuation binding, and cancels detached upstream work when the client disconnects. Resettable standard pings cover compact header waits and pre-visible Messages body silence without committing semantic output or blocking failover. Canceled clients do not penalize account health, and terminal state prevents duplicate error events after completion.
- **Upstream evidence:** Wei-Shaw/sub2api issue #3858 confirmed `/v1/messages` could swallow `response.failed` into HTTP 200 empty output. Merged PRs #3859 (`104fd2b6e`) and #3870 (`8f97953e5`) fixed core failure/failover semantics. Open PR #3808 contains related compact experiments but remains conflicting and was not wholesale imported. Issue #2064 independently reports HTTP 200/null content near the long-context threshold. PR #3906 covers native `/responses` compact rather than this Messages bridge, and issue #3961 reports a nil-writer failure in that path.
- **Evidence:** Representative unit reproductions cover terminal failure before visible output, reasoning-only output, missing terminal events, HTTP and SSE `context_length_exceeded`, full-history recovery, overflow usage accumulation, recursive split/merge limits, session-header removal, continuation suppression, terminal text/tool replay, stale tool delta rejection, header/body and pre-visible keepalive, post-ping SSE errors, duplicate-terminal suppression, and client-cancel propagation.
- **Regression tests:** `backend/internal/service/openai_gateway_messages_empty_output_test.go`, `backend/internal/service/openai_gateway_messages_compact_test.go`, `backend/internal/service/account_openai_compact_test.go`, `backend/internal/pkg/apicompat/anthropic_responses_test.go`, plus bridge/stream failover tests under `backend/internal/handler`.
- **Verification:** `go test -tags=unit ./internal/pkg/apicompat -count=1`; targeted service `Anthropic|Compact|Messages` tests; targeted handler bridge/stream tests; and complete `internal/service` plus `internal/handler` unit packages all pass with unrelated in-progress image-monitor files isolated through a temporary Go overlay.
- **Related:** Compact is still an upstream integration boundary. A client 4xx unrelated to context length remains a client error; retryable failures can switch accounts only before visible output is written; partial visible output cannot safely be replayed on another account. Standard pre-header `ping` keeps intermediaries alive, but newer Claude Code versions may only reset their internal watchdog from an empty delta after a real content block has begun; synthesizing such a block before recovery would make safe failover impossible.
- **Status:** DONE_WITH_CONCERNS. The deterministic failure shapes are fixed and locally verified, but the exact production conversation has not been replayed against the deployed environment, and no push or deployment was authorized in this task.
